Projet 2

La problématique est :
Comment appréhender les différents systèmes d’exploitation présents dans le monde actuel, et quels sont leur usages ?

Dans ce projet, nous avons vu l'ensemble du un système informatique, comment il est composé, qu’est-ce qu’un noyau, comment ça fonctionne, qu’est-ce qu’un système serveur et qu’est- ce qu’un système client, comment les deux fonctionnent de pair pour proposer des fonctionnalités indispensables au bon fonctionnement d’un ordinateur, d’une entreprise, d’un pays.

Nous avons vu 4 grands thèmes dans ce projet:

DNS

SI on veut accéder à un site web on va allez utiliser ce qu'on appelle un nom de domaine par exemple youtube.com. C'est le nom que vous tapez sur notre navigateur sauf que le serveur ou on veut y aller a besoin de l’adresse IP du site. Puisque l’ordinateur n’a pas besoin du nom de domaine mais de savoir quel serveur se connecté c’est à dire l’adresse IP au quelle il va chercher les page web qu’on cherche. Et l’adresse IP est tout simplement une suite de chiffre qui correspond à un serveur. Pour nous, une adresse IP c’est compliquer à retenir et on préfère retenir des mots plutôt qu’une suite de chiffre mais il y a un problème le navigateur n’a pas besoin de lettre pour accéder à la page web mais de chiffre. Et c’est là qu’intervient le DNS (domain name système) qui sert à traduire un nom de domaine (ex : google.fr) en adresse IP. Il sert de traducteur pour associer le nom qu’on marque sur le moteur de recherche à l’adresse IP.

RESEAU

Il faut d’abord savoir ce qu’est un réseau. Un réseau, en quelque mot, est un lien des ordinateurs et à des serveurs de communiquer efficacement au moyen d'un protocole de communication commun (IP). Le modèle OSI est un ensemble de règle qu’il faut suivre pour pouvoir faire fonctionner sur un réseau. Le modèle OSI est partagé en 7 couches. Et le modèle tcp/ip est une version simplifier du modèle OSI.

HTTP

Http (Hypertext Transfer Protocol) : C’est un langage informatique permet à l'utilisateur d'accéder à des pages web grâce à un navigateur (google, Firefox...). Fait la requête pour trouver l’information sue le web. C’est un échange entre navigateur web et l’utilisateur et c’est également la 4e couche du TCP/IP. Par exemple notre navigateur web est un client HTTP qui envoie des demandes à des serveurs.

OS

Pour qu’un système d’exploitation soit une interface utile et pratique entre l’utilisateur et le matériel, il doit fournir certains services de base, tels que la capacité de lire et d’écrire des fichiers, d’allouer et de gérer la mémoire, de prendre des décisions de c ontrôle d’accès, etc.

. Puis on ira sur l’autre de configuration pour créer et de configurer nos différente dans le fichier named.conf.local. On affiche notre nom de domaine et le chemin pour avoir notre zone de résolution directe. Ce paragraphe est la déclaration de la zone de recherche direct (chemin de notre fichier zone). La 1er permet de spécifier la zone de recherche Le second est la zone de recherche indirect. Ici il faut bien mettre de les 3 premier octet de notre serveur mais il faut les inverser.

Pour il faut les transmettre les fichiers déjà existant comme le fichier db.local qu’on va copier dans le fichier db.manu.com qui se trouve dans la zone de recherche direct. Une fois copier on configure. On remplace le “localhost” par l’FQDN c’est a dire le nom de la machine et de notre nom de domaine. Les 2 dernière ligne permet de voir les différents serveurs qu’on va utiliser l’hôte et le client. Apres il faut copier le ficher db.127 aux fichier db.192 qui se trouve dans la zone inverse. On fait pratiquement la même chose sauf qu’aux lieux de l’adresse ip c’est le nom du serveur et a la place des “@” on met le le nom de l’hote (la machine).
. Ensuite, il faut aller le fichier resolf.conf pour mettre le "nameserver" avec l'adresse ip pour moi c'était "192.168.1.80" et le "search ." le nom de notre dns, moi c'était "manu.com".
